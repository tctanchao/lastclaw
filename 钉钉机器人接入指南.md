## 方案概述

OpenClaw 目前不支持钉钉机器人，但可以通过以下两种方式接入：

| 方案 | 难度 | 维护成本 | 推荐度 |
|------|------|---------|--------|
| **开发通道插件** | ⭐⭐⭐ | 低 | ⭐⭐⭐⭐⭐ |
| **Webhook中转** | ⭐⭐ | 中 | ⭐⭐⭐ |

## 前置准备

### 1. 创建钉钉机器人

1. 打开钉钉群聊 → 群设置 → 智能群助手 → 添加机器人
2. 选择"自定义"机器人
3. 设置机器人名称和头像
4. 安全设置（三选一）：
   - **推荐：加签** - 生成密钥用于验证消息
   - 自定义关键词
   - IP地址白名单
5. 记录以下信息：
   - **Webhook URL**: `https://oapi.dingtalk.com/robot/send?access_token=xxx`
   - **加签密钥**: `SECxxx...`（如果选择了加签）

### 2. 开发环境要求

- Node.js 18+ 或 Node.js 20+
- TypeScript 基础
- 了解 OpenClaw 插件系统

## 方案一：开发通道插件（推荐）

### 架构设计

```
钉钉服务器 → 你的服务器（接收消息）→ OpenClaw Gateway → AI处理 → 返回钉钉
```

### 步骤 1：创建插件目录

```bash
mkdir -p ~/.openclaw/extensions/dingtalk
cd ~/.openclaw/extensions/dingtalk
```

### 步骤 3：创建插件主文件

创建 `index.ts`:

```typescript
import type { ChannelPlugin, ChannelPluginApi } from 'clawdbot/plugin-sdk';
import * as crypto from 'crypto';

// 钉钉消息类型
interface DingTalkMessage {
  msgtype: string;
  text?: { content: string };
  markdown?: { title: string; text: string };
  at?: { atMobiles?: string[]; atUserIds?: string[]; isAtAll?: boolean };
}

// 钉钉回调消息
interface DingTalkCallback {
  msgtype: string;
  content: string;
  createdAt: string;
  conversationId: string;
  senderNick: string;
  senderId: string;
  senderCorpId: string;
  senderDingtalkId: string;
  sessionWebhook: string;
  sessionWebhookExpiredTime: number;
}

// 验证签名
function verifySignature(timestamp: string, secret: string, sign: string): boolean {
  const stringToSign = `${timestamp}\n${secret}`;
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(stringToSign);
  const expectedSign = hmac.digest('base64');
  return sign === expectedSign;
}

// 生成签名
function generateSignature(timestamp: string, secret: string): string {
  const stringToSign = `${timestamp}\n${secret}`;
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(stringToSign);
  return encodeURIComponent(hmac.digest('base64'));
}

const plugin: ChannelPlugin = {
  id: 'dingtalk',
  meta: {
    id: 'dingtalk',
    label: '钉钉',
    selectionLabel: '钉钉机器人',
    docsPath: '/channels/dingtalk',
    blurb: '钉钉群聊机器人',
    aliases: ['dd', 'ding'],
  },

  capabilities: {
    chatTypes: ['group'],
    canSend: ['text', 'markdown'],
    canReceive: ['text'],
  },

  config: {
    listAccountIds: (cfg) => {
      return Object.keys(cfg.channels?.dingtalk?.accounts ?? {});
    },
    resolveAccount: (cfg, accountId) => {
      const accounts = cfg.channels?.dingtalk?.accounts ?? {};
      return accounts[accountId ?? 'default'] ?? { accountId };
    },
  },

  // HTTP网关，接收钉钉回调
  gateway: {
    start: async (api) => {
      const { httpServer, config } = api;

      // 注册HTTP处理器
      httpServer.post('/dingtalk/callback', async (req, res) => {
        try {
          const { timestamp, sign } = req.query as any;
          const body = req.body as DingTalkCallback;

          // 验证签名
          const accounts = config.channels?.dingtalk?.accounts ?? {};
          let verifiedAccount: string | null = null;

          for (const [accountId, account] of Object.entries(accounts)) {
            const acc = account as any;
            if (verifySignature(timestamp, acc.secret, sign)) {
              verifiedAccount = accountId;
              break;
            }
          }

          if (!verifiedAccount) {
            res.status(401).json({ error: 'Invalid signature' });
            return;
          }

          // 处理消息
          if (body.msgtype === 'text') {
            const message = {
              id: `dingtalk:${body.conversationId}:${Date.now()}`,
              channel: 'dingtalk',
              accountId: verifiedAccount,
              peer: {
                kind: 'group',
                id: body.conversationId,
              },
              sender: {
                id: body.senderId,
                name: body.senderNick,
              },
              body: body.content,
              timestamp: new Date(body.createdAt).getTime(),
              replyTo: null,
            };

            // 投递到 OpenClaw
            await api.deliverInbound(message);
          }

          res.json({ success: true });
        } catch (error) {
          api.logger.error('DingTalk callback error:', error);
          res.status(500).json({ error: 'Internal error' });
        }
      });

      api.logger.info('DingTalk channel started');
    },

    stop: async (api) => {
      api.logger.info('DingTalk channel stopped');
    },
  },

  // 发送消息
  outbound: {
    deliveryMode: 'direct',

    sendText: async ({ text, accountId, peer, config }) => {
      const account = config.channels?.dingtalk?.accounts?.[accountId ?? 'default'];
      if (!account) {
        return { ok: false, error: 'Account not found' };
      }

      const timestamp = Date.now().toString();
      const sign = generateSignature(timestamp, account.secret);

      const url = `${account.webhook}&timestamp=${timestamp}&sign=${sign}`;

      const message: DingTalkMessage = {
        msgtype: 'text',
        text: { content: text },
      };

      const response = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(message),
      });

      const result = await response.json();

      return {
        ok: result.errcode === 0,
        error: result.errmsg,
        messageId: `dingtalk:${peer.id}:${Date.now()}`,
      };
    },
  },

  // 健康检查
  status: async ({ config, accountId }) => {
    const account = config.channels?.dingtalk?.accounts?.[accountId ?? 'default'];
    if (!account) {
      return { healthy: false, message: 'Account not configured' };
    }

    // 发送测试消息（可选）
    return {
      healthy: true,
      message: 'DingTalk channel ready',
      details: {
        hasWebhook: !!account.webhook,
        hasSecret: !!account.secret,
      },
    };
  },
};

export default function register(api: ChannelPluginApi) {
  api.registerChannel({ plugin });
}
```

## 方案二：使用 Webhook 中转

适合快速测试，不需要开发插件。

### 架构

```
钉钉 → 中转服务 → OpenClaw Webhook → AI处理 → 返回钉钉
```

### 步骤 1：创建中转服务

创建 `dingtalk-bridge.js`:

```javascript
const http = require('http');
const crypto = require('crypto');

const PORT = 3000;
const OPENCLAW_URL = 'http://localhost:8787/api/message';
const DINGTALK_TOKEN = '你的access_token';
const DINGTALK_SECRET = 'SEC你的密钥';

// 验证签名
function verifySignature(timestamp, secret, sign) {
  const stringToSign = `${timestamp}\n${secret}`;
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(stringToSign);
  return sign === hmac.digest('base64');
}

// 生成签名
function generateSignature(timestamp, secret) {
  const stringToSign = `${timestamp}\n${secret}`;
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(stringToSign);
  return encodeURIComponent(hmac.digest('base64'));
}

// 发送消息到钉钉
async function sendToDingTalk(content) {
  const timestamp = Date.now().toString();
  const sign = generateSignature(timestamp, DINGTALK_SECRET);

  const url = `${DINGTALK_TOKEN}&timestamp=${timestamp}&sign=${sign}`;

  await fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      msgtype: 'text',
      text: { content }
    })
  });
}

// 发送消息到 OpenClaw
async function sendToOpenClaw(message) {
  await fetch(OPENCLAW_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      channel: 'webhook',
      sender: message.senderId,
      message: message.content,
      metadata: {
        source: 'dingtalk',
        conversationId: message.conversationId
      }
    })
  });
}

// 创建HTTP服务器
const server = http.createServer(async (req, res) => {
  if (req.method === 'POST' && req.url === '/callback') {
    let body = '';

    req.on('data', chunk => body += chunk);
    req.on('end', async () => {
      try {
        const data = JSON.parse(body);
        const { timestamp, sign } = req.query;

        // 验证签名
        if (!verifySignature(timestamp, DINGTALK_SECRET, sign)) {
          res.writeHead(401);
          res.end('Invalid signature');
          return;
        }

        // 处理消息
        if (data.msgtype === 'text') {
          // 发送到 OpenClaw
          await sendToOpenClaw({
            senderId: data.senderId,
            content: data.content,
            conversationId: data.conversationId
          });
        }

        res.writeHead(200);
        res.end('OK');
      } catch (error) {
        console.error('Error:', error);
        res.writeHead(500);
        res.end('Error');
      }
    });
  } else {
    res.writeHead(404);
    res.end('Not found');
  }
});

server.listen(PORT, () => {
  console.log(`DingTalk bridge running on port ${PORT}`);
  console.log(`Callback URL: http://your-server:${PORT}/callback`);
});
```


## 常见问题

### Q: 签名验证失败？

**A**: 检查以下几点：
- 时间戳是否正确（钉钉要求5分钟内）
- 密钥是否完整（包括 SEC 前缀）
- 编码方式是否正确（base64）

### Q: 收不到消息？

**A**: 检查：
- 回调地址是否可访问（公网IP）
- 端口是否开放
- 钉钉机器人是否配置了回调

### Q: 如何支持Markdown？

**A**: 修改 `sendText` 方法：

```typescript
sendText: async ({ text, accountId, peer, config }) => {
  // ...
  const message = {
    msgtype: 'markdown',
    markdown: {
      title: 'AI回复',
      text: text  // Markdown格式
    }
  };
  // ...
}
```

### Q: 如何支持@特定用户？

**A**: 添加 `at` 字段：

```typescript
const message = {
  msgtype: 'text',
  text: { content: text },
  at: {
    atUserIds: ['user-id'],  // 被@的用户ID
    isAtAll: false  // 是否@所有人
  }
};
```

### Q: 如何使用内网穿透？

**A**: 推荐工具：
- **ngrok**: `ngrok http 3000`
- **frp**: 自建内网穿透服务
- **钉钉 Stream 模式**: 无需公网IP（需要企业开发者权限）

## 进阶功能

### 1. 支持 Stream 模式

钉钉 Stream 模式可以避免使用公网IP：

```typescript
// 使用钉钉 Stream SDK
import { StreamClient } from '@dingtalk/dingtalk-stream';

gateway: {
  start: async (api) => {
    const client = new StreamClient({
      clientId: 'your-client-id',
      clientSecret: 'your-client-secret',
    });

    client.on('message', async (message) => {
      // 处理消息
    });

    await client.start();
  }
}
```

### 2. 支持富媒体

钉钉支持多种消息类型：

- **ActionCard**: 卡片消息
- **Link**: 链接消息
- **FeedCard**: 多条卡片消息

### 3. 群管理功能

- 获取群成员列表
- 群消息@所有人
- 设置群公告
