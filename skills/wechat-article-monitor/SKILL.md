# 公众号文章监控 📰

> ⚠️ **重要提示：这是一个技术演示版本，模拟数据仅用于展示 OpenClaw 技能开发框架。无法实际采集真实微信公众号数据。**

监控微信公众号最新文章，实时推送更新提醒（演示版本)

## 功能特性

### ✅ 核心功能（演示版）

- **添加公众号**: 支持通过公众号名称或 ID添加
- **文章采集**: 获取最新 N 篇文章（返回模拟数据）
- **实时监控**: 检测新文章发布并推送通知
- **数据存储**: 本地保存历史记录，避免重复提醒
- **文章导出**: 导出为 CSV/Markdown 格式
- **定时检查**: 可配置检查间隔（默认 1 小时）

### 🎯 使用场景

| 场景 | 说明 |
|------|------|
| 内容学习 | 监控行业大号，第一时间学习新内容 |
| 竞品分析 | 跟踪竞品公众号，分析发布策略 |
| 企业监控 | 监控公司公众号，及时获取行业动态 |
| 账号运营 | 监控自己的公众号，确认文章发布成功 |

## 安装

```bash
openclaw skill install wechat-article-monitor
```

## 配置

在 `~/.openclaw/config/skills/wechat-article-monitor.json` 中配置：

```json
{
  "checkInterval": 3600,
  "notifyEnabled": true,
  "maxArticles": 10,
  "apiKey": "your-wechat-api-key"
}
```

### 配置说明

| 参数 | 默认值 | 说明 |
|------|----------|------|
| checkInterval | 3600 | 检查间隔（秒），默认 1 小时 |
| notifyEnabled | true | 是否启用通知推送 |
| maxArticles | 10 | 每次获取的最大文章数 |
| apiKey | - | 微信 API Key（可选，演示版本暂未实现） |

## 技术实现

### 架构设计

```
┌─────────────┐
│  index.js   │  主入口，协调各模块
└──────┬──────┘
       │
  ┌────┴────┬────────┬────────┐
  ▼         ▼        ▼        ▼
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│Monitor│ │Storage│ │Notifier│ │ Cron │
└──────┘ └──────┘ └──────┘ └──────┘
```

### 模块说明

- **Monitor**: 负责数据采集（演示版本返回模拟数据）
- **Storage**: 负责数据持久化（本地 JSON 文件）
- **Notifier**: 负责发送通知（演示版本仅控制台输出）
- **Cron**: 负责定时任务调度
