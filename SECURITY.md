# 安全策略

## 支持的版本

我们只支持当前最新版本的演示版本（v0.1.0-demo）。

## 报告安全漏洞

如果你发现安全漏洞，请不要直接公开报告。请通过以下方式联系我们：

- **邮箱**: security@lastclaw.com
- **GitHub**: 使用 [GitHub Security Advisory](https://github.com/tctanchao/lastclaw/security/advisories/new)

请在报告中包含：
- 漏洞描述
- 复现步骤
- 影响范围
- 可能的修复方案

我们会在收到报告后尽快响应。

## 安全最佳实践

对于使用本技能的用户：

1. **不要提交敏感信息**
   - 不要在代码中包含 API 密钥
   - 不要在公共仓库中包含配置文件

2. **使用环境变量**
   - 将敏感配置存储在环境变量中
   - 使用 `.env` 文件（不要提交到 git）

3. **定期更新依赖**
   - 定期运行 `npm audit`
   - 及时更新有安全漏洞的依赖包

## 依赖安全

本项目使用 `npm audit` 来检查依赖项的安全问题。
